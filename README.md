# تطبيق مسار - المناديب (Android)

تطبيق أندرويد أصلي مبني بـ **Kotlin + Jetpack Compose** — ليس متصفحًا، بل واجهة أصيلة.

## مميزات التطبيق

### تسجيل الدخول لمرة واحدة فقط
- في أول تشغيل: المندوب يدخل **رابط الموقع** (مثل `vr2.umniate.com`)
- ثم يدخل **رقم الإقامة + كلمة المرور** التي ضبطتها له في «سجل المناديب»
- بعدها التطبيق **يحفظ التوكن للأبد** — لا يطلب البيانات مرة أخرى
- يستطيع الخروج يدويًا من «حسابي»

### واجهة بـ 4 تبويبات (Bottom Navigation)

| التبويب | المحتوى |
|---------|---------|
| 🏠 **نظرة عامة** | اسم المندوب، صورته، فئته، ترتيبه، طلباته، VDA، الكيلومترات |
| 🛠 **الخدمات** | شكوى / حادث / بنزين / إجازة / سلفة / تقرير |
| 📋 **طلباتي** | تاريخ كل الطلبات السابقة بحالتها (معتمد / مرفوض / بانتظار) |
| 👤 **حسابي** | تفاصيل الراتب، الخصومات، المكافآت، الصافي + زر الخروج |

### الأمان
- لا يحوي التطبيق بيانات قاعدة البيانات إطلاقًا
- يتصل عبر API فقط مع توكن HMAC آمن
- يعمل عبر HTTPS فقط

## كيف تحصل على APK جاهز (الطريقة الموصى بها)

### عبر GitHub Actions — مجاني، 5-7 دقائق

1. **أنشئ حساب** على [github.com](https://github.com) إن لم يكن لديك
2. اضغط زر **+** ← **New repository**
   - الاسم: `masar-app`
   - **اجعله Public** (مهم — Private يستهلك من حصتك المجانية)
   - اضغط **Create repository**
3. في الصفحة التالية اضغط على رابط **uploading an existing file**
4. **فك ضغط** `masar_android_app.zip` على جهازك
5. **افتح** المجلد المفكوك واختر كل ما بداخله (Ctrl+A)
6. **اسحب** الملفات إلى نافذة GitHub
7. انتظر اكتمال الرفع → اضغط **Commit changes**
8. اضغط على تبويب **Actions** في الأعلى
9. ستجد تشغيلاً جاريًا اسمه "Build Android APK" (دائرة صفراء)
10. انتظر 5-7 دقائق حتى يصبح أخضر ✓
11. اضغط على التشغيل → في الأسفل ستجد **Artifacts** → **MasarPortal-apk**
12. حمّل الملف، فك ضغطه، ستجد `app-debug.apk`

### عبر Android Studio — على جهازك

1. حمّل [Android Studio](https://developer.android.com/studio) (مجاني، ~8GB)
2. ثبّته (يأخذ ساعة تقريبًا أول مرة)
3. افتح مجلد المشروع المفكوك
4. انتظر حتى ينتهي Sync
5. اختر **Build → Build APK**
6. APK في `app/build/outputs/apk/debug/`

## كيف تثبت APK على جوال المندوب

1. أرسل ملف `app-debug.apk` للمندوب (واتساب/تيلجرام)
2. على جوال المندوب: افتح الملف
3. سيطلب الجوال السماح بـ **«تثبيت من مصادر غير معروفة»** — اضغط نعم
4. يثبت ويفتح
5. **عند أول تشغيل**: يدخل رابط الموقع (`vr2.umniate.com`)
6. **شاشة الدخول**: رقم الإقامة + كلمة المرور
7. بعدها يستخدم التطبيق مباشرة بلا تسجيل مرة أخرى

---

## ضبط المناديب في النظام قبل توزيع التطبيق

1. ادخل لوحة التحكم
2. **سجل المناديب** → اختر المندوب
3. في الأسفل ستجد قسم **«دخول تطبيق المندوب»**
4. اكتب كلمة مرور للمندوب (4 أحرف على الأقل)
5. اضغط حفظ
6. أعطِ المندوب: **رقم إقامته** + **كلمة المرور**

---

## التخصيص — تطبيق لشركة أخرى

من لوحة التحكم → **إدارة المستخدمين** → قسم العلامة التجارية:
- ارفع شعار جديد
- غيّر الاسم واللون الأساسي

هذه التغييرات **تطبّق فورًا** على شاشات التطبيق الداخلية. أما **أيقونة التطبيق** على جوال المندوب فهي مدمجة في APK — لتغييرها يجب بناء APK جديد بشعار جديد (استبدل `app/src/main/res/mipmap-*` ثم أعد البناء).

---

## في حال صادفت مشاكل في البناء

أرسل لي:
- لقطة شاشة من رسالة الخطأ في **Actions** (GitHub)
- أو سطر الخطأ في **Build** (Android Studio)

كل المكتبات والإصدارات في `app/build.gradle.kts` معروفة ومستقرة، لذا المشاكل عادة تكون في إعداد البيئة لا في الكود.
